# 좌표계

- 위치 척도와 상대적인 기하학적 배치의 조합을 뜻함.

# 직교 좌표계

![image.png](attachment:ab8f8356-dd11-4467-ba97-c1d3a1cf48b9:image.png)

- 데이터 시각화에 가장 널리 사용되는 좌표계로 2차원 직교 좌표계가 있음.
  - 각 위치는 x값과 y값으로 고유하게 지정됨
  - 두 축은 연속적인 위치 척도이므로 양수와 음의 실수 모두를 나타낼 수 있음
- 데카르트 좌표계는 서로 다른 단위를 나타내는 두 개의 축을 가질 수 있음.
  - 이를 이용해 서로 다른 유형의 변수를 x축과 y축에 매핑할 수 있다.
    ![image.png](attachment:a4cede0a-bd3f-4a1b-a8e0-d6b4c2dac72f:image.png)
    두 축의 단위가 다를 경우, 한 축을 다른 축에 대해 늘리거나 줄여도 데이터의 유효한 시각화를 유지할 수 있다.
    - 길고 좁은 그림은 y축을 따라 발생하는 변화를 강조
    - 짧고 넓은 그림은 그 반대의 효과를 줄 수 있음.
    ⇒ 중요한 위치 차이가 눈에 띄도록 적절한 종횡비를 선택하는 것이 좋음
  - x축과 y축의 단위가 같다면 두 축의 격자 간격이 같아야 한다. → 격자선이 정사각형
    ![image.png](attachment:db774917-f039-42dd-aff3-f2995c66ce55:image.png)

- 직교 좌표계는 선형 변환에 대해 불변이다.
  - 데이터의 단위를 변경하더라도, 축을 그에 맞게 변경하면 결과 그래프는 변하지 않는다.

# 비선형 축

- 직교 좌표계에서 사용되는 위치 스케일을 선형 스케일 이라고 함.
- 선형 스케일이 적합한 경우: 데이터 값을 있는 그대로 보여줄 때
- 비선형 스케일이 적합한 경우: 값의 범위 차이가 너무 크거나, 작은 변화가 중요할 때
  - 비선형 스케일에서는 데이터 간격과 그래프 간격이 비례하지 않을 수 있다.
    ![image.png](attachment:285b2aab-ac17-48f6-83f4-c7dbc1c2d439:image.png)
- 로그 스케일
  가장 일반적으로 사용되는 비선형 변환에는 로그 스케일이 있다. 로그 스케일을 사용하는 경우는 다음과 같다.
  1. 곱셈이나 나눗셈으로 얻은 데이터
     - 로그 스케일에서의 곱셈은 선형 스케일에서의 덧셈과 유사하기 때문
     - 특히 비율은 일반적으로 로그스케일로 표현.
  2. 데이터 세트에 크기가 매우 다른 숫자가 포함되어 있을 때
     - 하지만 데이터에 0값이 있다면 로그 변환 대신 제곱근 스케일을 사용하는 것이 바람직함.
       \*로그 스케일에서 $-\infin$에 위치하게 되기 때문이다.
- 제곱근 스케일
  제곱근 스케일은 로그 스케일보단 덜 일반적이다. 제곱근 스케일을 사용하는 경우는 다음과 같다.
  - 제곱 데이터가 있는 경우
    - 지리적 지역의 면적을 제곱근으로 나타내면 해당 지역의 동서 또는 남북 방향의 선형 범위를 강조할 수 있다. (2차원 값을 1차원 거리 개념으로 변환해서 보여주기 때문)
      → 특정 지역을 자동차로 횡단하는 경우 걸리는 시간을 계산할 때 유용
      ![image.png](attachment:5adefc6b-b5f5-48ca-906e-ff602b1c4581:image.png)

하지만 제곱근 스케일은 두 가지 문제점이 존재한다.

1. 제곱근 스케일에서는 단위의 의미가 없다.
   - 선형 스케일에서는 한 단위가 상수의 덧셈이나 뺄셈에 해당하고, 로그 스케일에서는 상수의 곱셈 또는 나눗셈에 해당함.
   - 제곱근 스케일에는 이러한 규칙이 존재하지 않음.
2. 축 눈금을 배치하는 것이 불분명함.

# 곡선축을 갖는 좌표계

1. 축 자체가 곡선인 좌표계에는 극 좌표계가 있다.

- 극 좌표계는 원점으로부터의 각도와 반지름 거리를 통해 위치를 나타냄.

  ![image.png](attachment:17329283-baa8-4ea9-b4b0-d7337370429b:image.png)

- 극 좌표계는 주기적인 특성을 가진 데이터에 유용하게 사용될 수 있다.
  - 이를 이용해 한쪽 끝의 데이터 값을 다른 쪽끝의 데이터 값과 논리적으로 연결할 수 있다. (12/31과 1/1)
  ![image.png](attachment:77b31c24-0c96-4067-93e5-fe692a645095:image.png)

1. 곡선 축을 접하는 또 다른 경우는 지리 공간 데이터가 있다.

   ![image.png](attachment:1691fcca-b74a-48a3-8b0e-420721fd73c7:image.png)

- 위도와 경도를 데카르트 시스템에 매핑하면 면적과 각도 측면에서 왜곡을 초래할 수 있다.
